# Project Meridian

**IBM × Memorial University watsonx Hackathon · Team Blue Meridian · May 29–31, 2026**

An agentic AI planner that charts Newfoundland and Labrador's path off diesel — for a single community, or all 20 at once under a budget.

## Structure

```
api/            FastAPI backend — agent tools, chat proxy, /briefs, /governance/report
agents/         Paste-ready watsonx Orchestrate agent definitions (7 agents)
data/           20 communities, 4 funding programs, generated briefs.json
frontend/       React + Vite + Tailwind — the demo surface (deployed to Vercel)
governance/     Model card + drift evaluation report
scripts/        run_batch.py (generates briefs.json), evaluate_briefs.py
tests/          pytest smoke tests for the API
```

## IBM watsonx products used

Meridian uses **all four** watsonx products provisioned for the hackathon, each doing a distinct job:

| Product | Role in Meridian |
|---|---|
| **watsonx.ai Runtime** | Granite inference for the chat and every agent (`api/chat.py`, `agents/*.md`). |
| **watsonx.ai Studio** (Prompt Lab) | Authoring and iterating the seven agent prompts in `agents/*.md`. |
| **watsonx Orchestrate** | Hosting the seven agents and the Layer 1 chat UI. |
| **watsonx.governance** | Drift evaluation (`scripts/evaluate_briefs.py`), model card (`governance/model_card.md`), audit trail (`governance/eval_briefs_report.json`). |

## Quick start

```bash
# 1. Python 3.11+
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. Configure credentials (chat uses these; data endpoints don't require them)
cp .env.example .env
# edit .env: IBM_CLOUD_API_KEY, WATSONX_PROJECT_ID, WATSONX_URL

# 3. Smoke-test the tools (no LLM required, no API keys needed)
pytest

# 4. Pre-compute the 20 community briefs
python scripts/run_batch.py --no-llm     # deterministic fallback (no Orchestrate needed)
# OR
python scripts/run_batch.py              # LLM-backed (requires Orchestrate set up)

# 5. Audit the briefs for hallucinated / drifted numbers (watsonx.governance)
python scripts/evaluate_briefs.py
# Writes governance/eval_briefs_report.json. Pass = 100% in fallback, ≥ 95% in LLM mode.

# 6. Run the API
uvicorn api.main:app --reload --port 8000
# OpenAPI spec at http://localhost:8000/openapi.json
# Swagger UI at http://localhost:8000/docs
# Health at http://localhost:8000/health

# 7. Run the frontend (separate terminal)
cd frontend
npm install
echo "VITE_API_BASE_URL=http://localhost:8000" > .env
npm run dev
# Open http://localhost:5173
```

## Deployment

See `DEPLOY.md`. Short version:
- **API** → your VM via `docker compose up -d --build`, fronted by Caddy for HTTPS.
- **Frontend** → Vercel (`npx vercel --prod` from `frontend/`), with `VITE_API_BASE_URL` set to your API's URL.

## Security notes

- `.env` holds API keys. Never commit. `.gitignore` covers it.
- All data sources are public (CER, NRCan, Pembina, NL Hydro PUB filings). No PII, no client data.
- `briefs.json` is generated locally and excluded from git.
- The FastAPI's CORS is `allow_origins=["*"]` — safe because the API is read-only over public data. Tighten for production.
- After the hackathon, rotate `IBM_CLOUD_API_KEY`.

See `architecture.md` §9 for the full security walk-through.
